# 🧠 What is Team Workflow?

Large projects are divided into smaller tasks.

Different team members work on:
- layouts
- components
- responsiveness
- animations
- APIs
- testing

GitHub helps teams coordinate all these tasks.

# 🗂️ What is GitHub Projects?

GitHub Projects is a task management system inside GitHub.

Think of it like:
- Trello
- Jira
- Kanban boards

Used directly inside GitHub.

# ⚠️ Important Team Rules

Always:
- communicate clearly
- pull latest changes
- test before pushing
- review code carefully

Never:
- break the main project
- delete teammates’ work
- ignore merge conflicts
- push random unfinished code

---

# 🧩 What are Merge Conflicts?

Merge conflicts happen when:
- two people edit the same file
- Git cannot decide which version to keep

They are normal in real projects.

Do not panic when they happen.
