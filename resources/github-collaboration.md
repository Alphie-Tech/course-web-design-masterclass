# 🤝 GitHub Collaboration – Alphie Tech Responsive Frontend Engineering Masterclass (10 Weeks)

Now that you understand GitHub basics, it is time to learn collaboration workflow.

Modern software development is a team activity.

Developers:
- review each other’s code
- suggest improvements
- work on shared projects
- collaborate using GitHub tools

---

# 🧠 What is Collaboration?

Collaboration means:
- multiple developers working on the same project
- reviewing code together
- improving code quality together

In this course, collaboration will happen through:
- peer review
- shared repositories
- pull requests
- team projects

---

# 🌿 What is a Branch?

A branch is an independent version of a project.

It allows you to:
- work safely
- experiment without affecting the main project

Think of it as:
## “your own workspace inside a project.”

---

# 🌱 Create a Branch

```bash
git checkout -b navbar-feature
```

---

# ✅ Check Current Branch

```bash
git branch
```

---

# 💾 Save Your Work

```bash
git add .
git commit -m "Added navbar section"
```

---

# 🚀 Push Branch to GitHub

```bash
git push origin navbar-feature
```

---

# 🔀 What is a Pull Request?

A Pull Request (PR) is a request asking:

> “Please review my code and merge it.”

This is how professional teams review code.

---

# 📋 Pull Request Workflow

1. Create branch
2. Write code
3. Commit changes
4. Push branch
5. Open Pull Request
6. Receive review
7. Fix issues if needed
8. Merge changes

---

# 👀 Code Review

Code review means:
- reading another developer’s code
- suggesting improvements
- spotting mistakes
- learning better techniques

This is one of the fastest ways to improve as a developer.

---

# 💬 Good Review Examples

✅ Good feedback:

```txt
Consider using semantic HTML here.
```

```txt
Nice responsiveness on mobile.
```

```txt
This variable name could be clearer.
```

---

# ❌ Bad Review Examples

❌ Avoid:

```txt
Bad code.
```

```txt
This is wrong.
```

Always be respectful and constructive.

---

# ⚠️ Collaboration Rules

Do NOT:
- overwrite other students’ work
- merge without review
- delete project files carelessly
- push unfinished broken code to main branches

---

# 🌟 Why Collaboration Matters

Companies hire developers who can:
- communicate
- collaborate
- review code
- work in teams

Technical skill alone is not enough.

---

# 🚀 Final Advice

Reading other people’s code improves your own coding ability.

Do not avoid peer review.

Embrace it.
