# 🌐 Responsive Frontend Engineering Masterclass (10 Weeks)

## 📅 Schedule & Commitment
- **Live Sessions:** Saturdays (2h) + Sundays (2h) = 4h/week
- **Independent Work:** Weekdays (~6–8h/week, 45–90 mins/day)
- **Submission Model:** Direct `git push` to your assigned repository
- **Core Focus:** Mobile-first architecture, accessibility, performance, documentation literacy, and professional Git workflows

---

## 📌 Philosophy
This course is not about memorizing syntax.  
It is about learning how to think like a frontend engineer:
- Break interfaces into logical, reusable components
- Debug systematically instead of guessing
- Design responsively with visual rhythm and consistency
- Read documentation fluently before searching for tutorials
- Collaborate professionally via Git and code reviews
- Build consistently under real deadlines

Aesthetics, accessibility, and engineering discipline are treated as equal priorities.

---

## 📥 Submission & Workflow Protocol

| Element | Standard |
|---------|----------|
| **Weekly Deadline** | **Friday at 11:59 PM** (local time) |
| **Git Workflow** | Weeks 1–5: Linear `main` only<br>Weeks 6–8: Feature branches → merge → `main`<br>Weeks 9–10: Team PR flow (shared repos) |
| **Commit Standard** | Conventional Commits: `feat:`, `fix:`, `refactor:`, `style:`, `chore:` |
| **Weekly Tag** | `v1` → `v10-final` (created & pushed every Friday) |
| **Review Focus** | Responsive behavior, accessibility, code structure, commit hygiene |
| **Accessibility Targets** | Full keyboard navigation, visible `:focus-visible` states, WCAG AA contrast, semantic structure, Lighthouse Accessibility ≥ 90 |

---

## 🗓️ 10-Week Curriculum

### 🔹 Week 1: HTML5, Git Basics & First Success
**Objectives:** Core markup, VS Code fluency, Git lifecycle, web architecture context  
**Live Sessions:** VS Code orientation, `index.html` anatomy, headings/links/images/lists, GitHub repo setup, `git init/add/commit/push`, viewport meta, semantic HTML, 20-min "How the Web Works" primer  
**Assignments:** Structured layout, semantic wrappers, accessible media, logical reading order validation  
**Deadline:** Friday 11:59 PM → Push `main`, tag `v1`  
**Milestone:** ✅ Semantic HTML Skeleton

### 🔹 Week 2: CSS3, Visual Rhythm & Browser Compatibility
**Objectives:** Professional UI taste, Flex/Grid, fluid units, cross-browser awareness  
**Live Sessions:** Box model, CSS reset, whitespace/alignment principles, responsive units (`rem`, `vw`, `clamp()`), mobile-first `@media`, browser fallbacks (`@supports`), UI Critique Session 1  
**Assignments:** Flexbox nav, CSS Grid layouts, breakpoint implementation, fluid typography, accessible form focus states  
**Deadline:** Friday 11:59 PM → Push `main`, tag `v2`  
**Milestone:** ✅ Professionally Styled, Cross-Browser Layout

### 🔹 Week 3: JavaScript, DOM & Debugging Mindset
**Objectives:** JS fundamentals, DOM manipulation, event handling, systematic debugging  
**Live Sessions:** DOM APIs, event delegation, `window.matchMedia`, DevTools deep dive (console, elements, network), binary isolation debugging  
**Assignments:** Dynamic element manipulation, event delegation practice, viewport-aware JS toggles, accessible form validation  
**Deadline:** Friday 11:59 PM → Push `main`, tag `v3`  
**Milestone:** ✅ Interactive Portfolio (JS + DevTools Fluency)

### 🔹 Week 4: Modern JS Refresher & React Fundamentals
**Objectives:** Pre-React ES6, JSX, components, props, `useState`, React debugging  
**Live Sessions:** `map/filter`, destructuring, ES modules, arrow functions, Vite + React, JSX rules, component hierarchy, error overlay reading  
**Assignments:** React initialization, component decomposition, prop drilling, reusable UI components, **Refactor Lab:** Clean up intentionally messy component  
**Deadline:** Friday 11:59 PM → Push `main`, tag `v4`  
**Milestone:** ✅ Componentized React Portfolio

### 🔹 Week 5: Hooks & Mid-Course Standalone Challenge
**Objectives:** `useEffect`, `useContext`, custom hooks, responsive state, independent build  
**Live Sessions:** `useEffect` lifecycle, dependency arrays, Context API, custom hooks (`useViewport`, `useLocalStorage`), theme sync, mini-project brief  
**Assignments:** State isolation, theme persistence, `prefers-color-scheme` sync, **Mid-Course Mini-Project:** Responsive Pricing Section or Weather Widget  
**Deadline:** Friday 11:59 PM → Push `main`, tag `v5`  
**Milestone:** ✅ Standalone Module + Portfolio State Layer

### 🔹 Week 6: Design Tokens, TailwindCSS & AI Introduction
**Objectives:** Design systems, utility-first styling, custom config, AI as accelerator  
**Live Sessions:** Spacing scales, typography rhythm, component consistency, Tailwind setup, `md:`/`lg:`/`dark:` prefixes, AI prompt basics, UI Critique Session 2  
**Assignments:** Token mapping, CSS→Tailwind migration, responsive utilities, config extension, arbitrary value avoidance  
**Deadline:** Friday 11:59 PM → Merge branch → `main`, tag `v6`  
**Milestone:** ✅ Utility-First Portfolio

### 🔹 Week 7: Framer Motion & Restraint in Animation
**Objectives:** Motion fundamentals, layout animations, scroll triggers, performance budgeting  
**Live Sessions:** `motion` components, `variants`, `useScroll`/`useInView`, gesture controls, reduced-motion fallbacks, UI Critique Session 3  
**Assignments:** Staggered reveals, scroll-triggered animations, `prefers-reduced-motion` logic, performance-conscious motion  
**Deadline:** Friday 11:59 PM → Merge branch → `main`, tag `v7`  
**Milestone:** ✅ Polished Animated Portfolio

### 🔹 Week 8: AI Optimization & Performance QA
**Objectives:** Advanced AI workflows, Lighthouse, bundle optimization, responsive QA  
**Live Sessions:** Prompt patterns for refactoring, Lighthouse auditing, `React.lazy()` + `Suspense`, cross-device QA, deployment prep  
**Assignments:** Advanced AI prompts, Lighthouse/axe fixes, code splitting, keyboard & accessibility audit, cross-browser validation  
**Deadline:** Friday 11:59 PM → Merge branch → `main`, tag `v8`  
**Milestone:** ✅ Optimized, Production-Ready Portfolio

### 🔹 Week 9: Team Project, Contracts & PR Flow
**Objectives:** GitHub flow, PRs, code review, team collaboration standards  
**Live Sessions:** Team repo onboarding, role assignment, PR templates, conflict resolution, code review rubrics  
**Assignments:** Team contract signing, issue creation, feature branch development, peer PR review, feedback iteration  
**Deadline:** Friday 11:59 PM → Team merges all PRs → `main`, tag `v9-team`  
**Milestone:** ✅ Team Project Feature Complete + Contract Fulfilled

### 🔹 Week 10: Deployment, Web Architecture & Career Prep
**Objectives:** Deployment confidence, CI/CD concepts, rollback, professional documentation  
**Live Sessions:** Client/server model, DNS/hosting, build logs, env vars, auto-deploy mechanics, rollback strategies, live demos, career prep  
**Assignments:** Final responsive polish, performance/a11y validation, deployment, **Professional README** (screenshots, stack, features, setup, live link)  
**Deadline:** Friday 11:59 PM → Push `main`, tag `v10-final`  
**Milestone:** ✅ Portfolio Deployed & Professionally Documented

## 🏗️ Projects

- **Requirements:** Mobile-first (320px–1440px+), `prefers-color-scheme` sync, keyboard navigability, visible focus states, semantic structure, Lighthouse ≥ 90, conventional commits

- **Features:** Landing page (Hero, Features, Pricing, FAQ, CTA) + Dashboard component (data table, filters, responsive grid, loading states)
- **Requirements:** Minimum 3 PRs per student, 2+ substantive peer review comments per PR, role declaration, `CONTRIBUTIONS.md` tracking
- **Deliverable:** Merged `main`, deployed site, team contract compliance

### 🎯 Mid-Course Mini-Project (Week 5)
- **Scope:** Independent build of a responsive Pricing Section or Weather Dashboard Widget
- **Requirements:** React + Hooks + Props + CSS/Tailwind, clean component architecture, state management
- **Purpose:** Validate foundational skills before larger refactors and team collaboration
